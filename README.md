# TRACE

**Transparent Review And Cryptographic Evidence**
*A Modular Cryptographic Peer Review Infrastructure*


TRACE is an open source protocol that separates technical rigor from ideological affinity in the evaluation of public content — and makes that separation permanently verifiable by anyone.

The first use case is journalism. The architecture is domain-agnostic.

---

## The problem

Public content — news articles, clinical studies, policy reports — is evaluated today through opaque, centralized, and manipulable systems. Fact-checkers are perceived as biased. Peer review is anonymous and untraceable. Editorial quality scores collapse methodology and ideology into a single number that serves no one honestly.

TRACE does not replace these systems. It adds a layer that none of them have: **a public, immutable, cryptographically traceable record of who said what about what, and when.**

---

## How it works

Every piece of content submitted to a TRACE instance receives two independent evaluations from verified experts:

```
TECHNICAL RIGOR     Did this work meet quality standards?
                    YES  /  NO  /  Insufficient context to judge

EDITORIAL AFFINITY  Do you agree with the thesis or approach?
                    YES  /  NO  /  Abstain          (optional)
```

These two dimensions are never collapsed into a single score. A reviewer can validate the rigor of work they disagree with. That combination — `Rigor: YES` + `Affinity: NO` — is the most valuable signal the system produces. It means an expert recognized quality across ideological lines. That vote is public, signed, and immutable.

The reader sees both dimensions separately, segmented by reviewer profile, and decides what to make of them. **TRACE does not interpret. It shows. The reader decides.**

## Architecture

TRACE is built as five independent modules. Any institution can adopt them separately or in combination, integrated over existing infrastructure via API.

```
┌──────────────────────────────────────────────────────────────┐
│  MODULE 1 — Cryptographic traceability                       │
│  Merkle Tree · GitHub · OpenTimestamps (Bitcoin) · Polygon   │
│  "What was said cannot be erased"                            │
├──────────────────────────────────────────────────────────────┤
│  MODULE 2 — Double vote system                               │
│  Rigor vs. affinity · Public · Signed · Immutable            │
│  "Separates the method from the message"                     │
├──────────────────────────────────────────────────────────────┤
│  MODULE 3 — Expert verification                              │
│  Objective criteria + peer endorsement + percentile ranking  │
│  "Who can evaluate and with what credibility"                │
├──────────────────────────────────────────────────────────────┤
│  MODULE 4 — Quality checklist                                │
│  Author-declared · Community-validated · Living document     │
│  "What criteria each domain defines"                         │
├──────────────────────────────────────────────────────────────┤
│  MODULE 5 — Visualization and public feed                    │
│  Progressive detail layers · No recommendation algorithm     │
│  "How the reader sees the information"                       │
└──────────────────────────────────────────────────────────────┘
```

Module 1 is the foundation. Any other module can be adopted independently. A platform that already has its own expert verification system can adopt only Modules 1 and 2. An institution that only needs immutable traceability over its existing rating system can adopt Module 1 alone.

---

## Core guarantees

These three properties apply to every instance of TRACE, regardless of domain or configuration. They are non-negotiable.

**Immutability.** What is registered cannot be modified retroactively without mathematical detection. Every vote is cryptographically signed by the reviewer and anchored daily in Bitcoin and Polygon blockchain. If the project closes tomorrow, the evaluation history survives indefinitely.

**Transparency.** The code that implements the system is fully public and auditable. There are no hidden algorithms, no opaque scoring weights, no editorial decisions made by the platform. A public script allows anyone to verify any evaluation without depending on the system.

**Portability.** Data generated by any instance can be exported in standard formats at any time. No institution is locked in. The immutable log remains in the public repository and blockchain regardless.

---

## Domain agnostic

TRACE's first instance is built for journalism. The protocol is designed for any domain where:

- Public content requires technical quality evaluation
- A verifiable expert community exists to evaluate it
- There is value in separating methodological rigor from ideological or commercial affinity
- Traceability of who said what matters

**Identified use cases:**

| Domain | What it solves |
|---|---|
| Journalism | Separates technical rigor of reporting from editorial affinity |
| Clinical medicine | Separates methodological quality of studies from controversial implications |
| Science communication | Verifies whether popular articles accurately represent the papers they cite |
| Policy reports / think tanks | Evaluates methodological rigor independently of ideological conclusions |
| Independent scientific journals | Transparent, traceable, public peer review |

---

## This is a protocol, not a platform

TRACE provides the 70% — the technical core, the traceability architecture, the vote system, the peer verification model, the philosophy of radical transparency. The remaining 30% — domain-specific checklist, specialization taxonomy, verification criteria — is completed by each community that adopts the system.

Governance is by fork. There is no committee, no mandatory RFC process, no negotiation over core principles. The creator defines and maintains the core with full transparency. Anyone who disagrees has the code and can do whatever they want with it. That is the AGPL license working exactly as designed.

Fragmentation is not a defect. It is the adaptation mechanism.

## Cost of operation

The traceability infrastructure costs approximately **$0.73/year** in Polygon gas fees. Everything else — GitHub, OpenTimestamps, the public verification script — is free. The only real operational cost is the backend server (~$5-10/month on Railway or Render).

## Embedable widget and API

> Planned for Phase 2. Not in MVP.

Any platform can integrate TRACE evaluations without leaving their own site.

**One line of code:**
```html
<script src="https://cdn.trace-protocol.org/widget.js"
        data-content-url="https://example.com/article"
        data-mode="standard">
</script>
```

**REST API per module** for platforms that want to consume specific modules over their existing infrastructure without adopting the full system.

## Philosophy

TRACE treats readers and experts as functional adults. It is not a parent. It is not an arbiter. It is not an authority.

It gives information. What anyone does with it is their decision.

If the platform is not used, that is the problem of those who do not use it. The infrastructure is here for whoever wants it.
